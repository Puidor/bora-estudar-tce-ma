Linguagem DML (Manipulação de Dados)

⚡ Direto ao Ponto

  • O que faz? A DML (Data Manipulation Language) é responsável por INSERIR, ALTERAR E APAGAR os registros (linhas) de uma tabela já existente.
  • Não manipula a estrutura: Se você quiser adicionar uma nova coluna ou apagar a tabela inteira do sistema, você deve usar a DDL (ALTER, DROP).
  • O Poder do Arrependimento: Comandos DML geram logs de transação precisos. A menos que a transação seja "comitada", você pode usar o comando ROLLBACK para desfazer as alterações e voltar no tempo.

🛠️ O Trio da Manipulação (DML)

INSERT

Insere novos registros. Permite adicionar uma nova linha preenchendo as colunas com os valores correspondentes.

UPDATE

Atualiza registros já existentes. Usado para trocar um valor antigo por um novo (ex: aumentar salário de quem bater a meta).

DELETE

Apaga registros da tabela. Pode apagar uma linha, várias ou todas (se você não usar um filtro), mantendo a estrutura da tabela intacta.

🔥 O Perigo Número 1 (A Regra de Ouro)

UPDATE clientes SET status = 'INATIVO';

Sem o WHERE, TODOS os clientes do banco de dados ficarão inativos ao mesmo tempo.

VS

UPDATE clientes SET status = 'INATIVO'
WHERE id_cliente = 5;

Com o WHERE, APENAS o cliente 5 terá o seu status alterado de forma segura.

A mesma regra de destruição em massa aplica-se ao DELETE sem WHERE!

🚨 Pegadinhas Mortais de Sintaxe

  • A Ilusão do Asterisco no DELETE: A banca vai te dar o código DELETE * FROM tabela WHERE id = 1; e perguntar se está correto.
    Falso! Diferentemente do SELECT *, o comando DELETE exclui a linha inteira por definição. Portanto, não se usa asterisco no comando DELETE (o correto em ANSI SQL é simplesmente DELETE FROM tabela).
  • Ocultar nomes das colunas no INSERT: "É obrigatório declarar o nome de todas as colunas no comando INSERT INTO".
    Falso! Você pode omitir a lista de colunas, DESDE QUE passe na cláusula VALUES os dados exatos para todas as colunas da tabela e na mesma ordem em que elas foram criadas originalmente.

1. A Sintaxe do INSERT

O comando INSERT INTO é utilizado para adicionar novos registros numa tabela. Há duas formas principais de escrever este comando, o que é frequentemente cobrado para testar a sua atenção a detalhes.

Forma Explícita (Recomendada)

Você nomeia as colunas em que deseja inserir dados. Isso permite inserir dados fora de ordem e omitir colunas que aceitem NULL ou que tenham valores DEFAULT.

INSERT INTO funcionarios (nome, salario, departamento)
VALUES ('João Silva', 5000, 'TI');

Forma Implícita (Perigosa)

Se omitir os nomes das colunas, tem de inserir um valor para cada coluna que existe na tabela, estritamente na ordem física em que a tabela foi desenhada.

-- Assumindo que a tabela tem id, nome, salario, departamento, nesta ordem:
INSERT INTO funcionarios
VALUES (101, 'Maria Sousa', 6000, 'RH');

2. A Sintaxe do UPDATE

O comando UPDATE altera dados que já estão na tabela. A palavra-chave SET é usada para indicar qual coluna receberá o novo valor.

É perigosíssimo esquecer a cláusula WHERE, pois o comando afetará a tabela inteira!

-- Atualizando apenas um registro:
UPDATE produtos
SET preco = 150.00
WHERE id_produto = 10;

-- Atualizando múltiplas colunas de uma vez:
UPDATE empregados
SET cargo = 'Gerente', salario = 8000
WHERE id_empregado = 45;

3. A Sintaxe do DELETE e sua diferença para DDL

O DELETE apaga as linhas da tabela. A estrutura da tabela (colunas, tipos) e os próprios metadados não são afetados.

-- Apaga apenas os inativos:
DELETE FROM usuarios WHERE status = 'inativo';

-- Apaga TUDO (Lento, mas com log de transação):
DELETE FROM usuarios;

Relembrando: DELETE vs TRUNCATE (Item 2.9 do Edital)

Embora o DELETE FROM tabela (sem WHERE) apague todos os registros, assim como o TRUNCATE TABLE tabela, a banca exigirá que você diferencie a mecânica interna:

  • O DELETE é DML: Ele vai linha a linha registrando no log que a apagou. Acionará triggers (gatilhos) de exclusão e poderá ser desfeito via ROLLBACK.
  • O TRUNCATE é DDL: Ele redefine o ponteiro da tabela, esvaziando-a imediatamente (muito mais rápido). Não aciona triggers de exclusão e geralmente efetua commit automático, impedindo o ROLLBACK.

4. DML e Transações (TCL)

Uma das características mais fortes da DML em bancos de dados relacionais é o respeito profundo pelas Transações (Propriedades ACID).

Quando você digita um UPDATE ou INSERT, na vasta maioria dos SGBDs modernos (em modo transacional), essa mudança ocorre apenas na memória RAM e nos seus logs de sessão. Para o resto da empresa, o banco de dados continua mostrando o valor antigo.

  • Para efetivar e "cravar" a mudança definitivamente no disco, tornando-a visível a todos, você precisa utilizar o comando de Controle de Transação: COMMIT.
  • Se algo der errado, se cair a energia, ou se você perceber que fez um UPDATE sem WHERE a tempo, você pode usar o comando ROLLBACK para desfazer a alteração (apenas as que ainda não sofreram Commit).

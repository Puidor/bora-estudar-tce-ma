6. Integridade Referencial

🔗 A Regra de Ouro ("Não Órfão")

A Integridade Referencial garante que os relacionamentos entre tabelas permaneçam consistentes.

Em termos práticos: O valor de uma Chave Estrangeira (FK) na Tabela "Filha" DEVE OBRIGATORIAMENTE existir como um valor na Chave Primária (PK) da Tabela "Pai"... Ou Ser Nulo (NULL).

⚠️ O que acontece se eu apagar o "Pai"?

Regras aplicadas com ON DELETE ou ON UPDATE na Chave Estrangeira.

RESTRICT (Default)

Bloqueia a ação. O banco de dados dá erro e impede a exclusão do registro Pai se houver algum Filho apontando para ele. (Segurança máxima).

CASCADE (Cascata)

Efeito Dominó. Se apagar o Pai, o banco de dados apaga automaticamente todos os Filhos associados. Útil para entidades fracas (Ex: Apagar o "Pedido" apaga os "Itens do Pedido").

SET NULL

Orfanato Seguro. Se apagar o Pai, o banco não apaga o Filho, mas sim transforma a FK do Filho em NULL. (Atenção: A coluna da FK não pode estar marcada como NOT NULL).

🚨 Top Pegadinhas de Prova

  • A Chave Estrangeira pode ser Nula (NULL)?
    A banca diz que é terminantemente proibido. Falso! A regra da Integridade Referencial permite valores nulos na FK. Exemplo: A tabela Funcionário tem a FK ID_Departamento. Um funcionário recém-contratado pode ter o departamento como NULL até ser alocado.
  • Autorrelacionamento:
    "Uma Chave Estrangeira deve obrigatoriamente apontar para uma tabela diferente". Falso! A FK pode apontar para a PRÓPRIA tabela. Exemplo Clássico: Tabela Funcionário onde a FK ID_Chefe aponta para o ID_Funcionario na mesma tabela.

1. Fundamentos da Integridade Relacional

No Modelo Relacional (criado por Edgar F. Codd), a integridade dos dados é protegida por regras (restrições) garantidas pelo SGBD. Existem três tipos principais de integridade:

  • Integridade de Entidade: Nenhuma Chave Primária (PK) pode ser Nula. Garante que cada linha seja identificável de forma única.
  • Integridade Semântica (Domínio): Os valores de uma coluna devem respeitar o tipo de dado e as restrições impostas. (Ex: Idade não pode ser "-5").
  • Integridade Referencial: É o "cimento" que une o banco de dados. Garante que as Chaves Estrangeiras (Foreign Keys) apontem sempre para uma referência válida ou apontem para "nada" (NULL).

2. A Mecânica da Chave Estrangeira (FK)

A Chave Estrangeira (FK) é um atributo (ou conjunto de atributos) de uma tabela cuja função é ser a referência para a Chave Primária (PK) de outra tabela (ou da mesma).

A Regra do Mapeamento

Segundo a teoria relacional estrita: Toda Chave Estrangeira deve referenciar uma Chave Primária (PK). No entanto, é importante saber para provas de SQL avançado que a maioria dos SGBDs comerciais permite que a FK referencie qualquer coluna que possua uma restrição de UNIQUE (Chave Alternativa), garantindo a unicidade.

Tratamento de Inserções e Exclusões

A integridade referencial impõe barreiras nos Comandos DML (Insert, Update, Delete):

  • Na Inserção (Tabela Filha): Você não pode inserir na tabela filha (ex: Venda) um registro referenciando um ID_Cliente = 99 se o cliente 99 não existir na tabela pai (Cliente).
  • Na Exclusão (Tabela Pai): Você não pode deletar o Cliente 10 se ele já tiver compras cadastradas na tabela filha (Venda), a menos que você configure uma regra especial de cascata.

3. Clausulas ON DELETE e ON UPDATE

Para resolver o problema da "Exclusão na Tabela Pai", os SGBDs oferecem modificadores ao criar a Chave Estrangeira no DDL. As bancas exigem que você saiba o comportamento exato de cada um:

Comando DDL Ação do Banco de Dados
RESTRICT (ou NO ACTION) O comportamento padrão. Rejeita a exclusão ou atualização do Pai se houverem referências no Filho. Dispara um erro de Foreign Key Constraint.
CASCADE Se o registro Pai for alterado ou deletado, a mudança propaga-se automaticamente para todos os registros Filhos que o referenciam.
SET NULL Exclui o registro Pai normalmente. Para não deixar os Filhos "órfãos apontando pro nada", ele troca o valor da Chave Estrangeira nos Filhos para NULL.
SET DEFAULT Exclui o registro Pai. Os Filhos que apontavam para ele têm a sua Chave Estrangeira alterada para um valor Padrão (Default) previamente configurado na criação da tabela.

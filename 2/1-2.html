Dados Abertos

⚡ Conceito Nuclear

  • O que são? Dados que podem ser livremente usados, reutilizados e redistribuídos por qualquer pessoa, sujeitos, no máximo, à exigência de atribuição da fonte e compartilhamento pelas mesmas regras.
  • Não basta estar "na internet": Para ser um Dado Aberto Governamental, não basta publicar um PDF no site. O dado precisa estar num formato aberto e processável por máquina.

🚨 Pegadinhas de Prova (Cuidado!)

  • Dados Abertos vs LGPD (Dados Pessoais): A banca diz que "Por ser um dado aberto governamental (ex: salário de servidor), a LGPD não se aplica".
    Falso! A publicação de dados abertos pelo governo deve respeitar a privacidade. Dados que identifiquem cidadãos comuns devem passar por anonimização antes da abertura. A transparência do Estado não anula o direito à privacidade do indivíduo.
  • Acesso vs Aberto: "Se está publicado num site e qualquer um pode ler, então é um dado aberto".
    Falso! Estar público (acessível) não é o mesmo que ser aberto. Uma imagem JPG de um balancete escaneado é público, mas fere os princípios de "Processável por Máquina" e "Primário".

1. Fundamentos e a "Open Definition"

O conceito de Dados Abertos (Open Data) transcende a simples transparência. Trata-se de disponibilizar dados de forma que a sociedade, pesquisadores e outras empresas possam criar novos produtos, softwares e análises em cima dessa informação, gerando valor econômico e social.

Segundo a Open Knowledge Foundation, "dados são abertos quando qualquer pessoa pode livremente acessá-los, utilizá-los, modificá-los e compartilhá-los para qualquer finalidade".

2. Os 8 Princípios dos Dados Abertos Governamentais

Em 2007, um grupo de ativistas e especialistas definiu os 8 princípios fundamentais. A memorização das definições exatas de cada um é matéria obrigatória para concursos:

  • 1. Completos: Todos os dados públicos devem ser disponibilizados, exceto aqueles sujeitos a restrições válidas de privacidade, segurança ou controle de acesso.
  • 2. Primários: Os dados devem ser publicados na fonte, com a maior granularidade possível (nível mais detalhado), e não de forma agregada ou modificada (ex: publicar as transações diárias, e não apenas o somatório mensal).
  • 3. Atuais: Devem ser disponibilizados o mais rápido possível para preservar o seu valor (evitar dados obsoletos).
  • 4. Acessíveis: Disponibilizados para o maior número possível de usuários e finalidades, preferencialmente via internet (download direto ou APIs).
  • 5. Processáveis por máquina: O dado deve ser estruturado o suficiente para permitir o processamento automatizado por softwares. (Aqui é onde o PDF falha miseravelmente).
  • 6. Acesso não discriminatório: Não pode haver exigência de cadastro, registro ou identificação do usuário para baixar o arquivo (sem catracas virtuais).
  • 7. Não proprietários: Devem estar em formatos sobre os quais nenhuma entidade tenha controle exclusivo. (Atenção: Planilhas XLS são de controle da Microsoft. O formato correto é o CSV ou ODS).
  • 8. Livres de licenças: Não devem estar sujeitos a direitos autorais, patentes, marcas registradas ou regulações de segredo industrial.

3. Modelo de 5 Estrelas (Tim Berners-Lee)

O criador da Web propôs um esquema de classificação para medir a qualidade dos Dados Abertos. Isso é um diferencial para provas avançadas:

  • ★ Disponível na Web com licença aberta (qualquer formato, ex: um PDF escaneado).
  • ★★ Disponível como dados estruturados e processáveis por máquina (ex: uma tabela em formato proprietário como o Excel .xls).
  • ★★★ Disponível num formato não proprietário (ex: CSV em vez de Excel). É o mínimo aceitável para bons portais de transparência.
  • ★★★★ Utiliza URIs (padrão W3C) para identificar as coisas, permitindo que as pessoas apontem diretamente para seus dados (Linked Data básico).
  • ★★★★★ Vincula seus dados aos de outras pessoas para fornecer contexto (Linked Open Data - LOD). O auge da utilidade.

4. Dados Abertos e Privacidade (Conflito com LGPD)

A publicação de Dados Abertos Governamentais exige um equilíbrio delicado com a LGPD (Lei Geral de Proteção de Dados Pessoais).

A regra geral é: o governo deve abrir os dados referentes à execução orçamentária e administrativa (Lei de Acesso à Informação). No entanto, se o dataset (conjunto de dados) contiver informações sobre cidadãos (como uma lista de pacientes de um hospital público ou inscritos num programa social), esses dados não podem ser expostos abertamente de forma a identificar os indivíduos.

A solução técnica da Engenharia de Dados é o processo de Anonimização. Técnicas como a remoção de colunas identificadoras (Nome, CPF), generalização (trocar a data de nascimento apenas pelo ano) ou perturbação de dados são aplicadas para que o dado mantenha o seu valor estatístico (pesquisa), mas seja irreversível quanto à identificação da pessoa física.

💾 Os Três Tipos de Armazenamento

Objeto (Object)

Ex: Amazon S3

  • • Dados são guardados como "Objetos" em "Buckets".
  • • Não há hierarquia de pastas real, tudo é plano.
  • • Perfeito para mídias, backups e big data. Acesso via APIs (HTTP/REST).
  • • Não serve para instalar Sistema Operacional!

Bloco (Block)

Ex: Amazon EBS

  • • Funciona como um HD tradicional plugado na placa-mãe.
  • • Dividido em blocos de tamanho fixo de alto desempenho.
  • • Só pode ser atachado a uma VM (Instância) por vez (geralmente).
  • • É aqui que se instala o S.O. (Windows/Linux) e o BD.

Arquivo (File)

Ex: Amazon EFS

  • • O clássico servidor de arquivos de rede (NAS).
  • • Hierarquia de diretórios (pastas e subpastas).
  • • Pode ser montado (compartilhado) por várias VMs ao mesmo tempo via NFS ou SMB.

🚨 Pegadinhas Mortais da Banca (CEBRASPE)

  • S3 como Disco Rígido:
    A banca diz: "O administrador de redes pode instanciar uma máquina EC2 na AWS e instalar o sistema operacional Ubuntu diretamente em um bucket do Amazon S3 para garantir durabilidade."
    Falso! O Amazon S3 é armazenamento de OBJETOS. Não possui sistema de arquivos capaz de realizar *boot* de sistema operacional. Para instalar um S.O., é obrigatório o uso de armazenamento em BLOCO (Amazon EBS).
  • PaaS x Serverless:
    "O Amazon EC2 é um exemplo de serviço serverless, pois isenta o usuário de gerenciar as instâncias físicas de servidores."
    Falso! O Amazon EC2 é a quintessência do IaaS (você tem que escolher RAM, CPU, instalar S.O. e gerenciar tudo). Um exemplo de Serverless na AWS seria o AWS Lambda.
  • Disponibilidade vs Durabilidade no S3:
    Cuidado com o jogo de palavras. A AWS garante "11 noves" (99.999999999%) de DURABILIDADE no S3 (chance de não perder seu arquivo ao longo de um ano). A DISPONIBILIDADE (chance de você conseguir acessar o arquivo num dado momento) costuma ser de 99.9% a 99.99%, dependendo da classe de armazenamento.

1. Amazon Web Services (AWS)

A AWS é a subsidiária de nuvem da Amazon. Sendo a pioneira (lançada em 2006), detém a maior fatia de mercado e é o referencial de nomenclatura para as bancas de concurso. A maioria das questões genéricas sobre nuvem terá alternativas contendo serviços da AWS.

Principais Serviços que caem em prova:

  • Amazon EC2 (Elastic Compute Cloud): O principal serviço IaaS da AWS. Fornece capacidade computacional escalável. Você aluga uma Máquina Virtual (chamada de Instância), escolhe a Amazon Machine Image (AMI) como o Ubuntu ou Windows Server e tem acesso *root* ao SO.
  • Amazon S3 (Simple Storage Service): Armazenamento de objetos projetado para arquivar e recuperar qualquer quantidade de dados da web. Os arquivos ficam em Buckets. É excelente para imagens de sites, backups estruturados e hospedar sites estáticos (HTML/CSS puros).
  • Amazon RDS (Relational Database Service): Serviço PaaS que facilita a configuração, operação e escalonamento de um banco de dados relacional. Suporta MySQL, PostgreSQL, Oracle, SQL Server e o banco próprio da Amazon: Aurora. O cliente não precisa aplicar *patches* no Linux do servidor do banco, a AWS faz isso.
  • AWS Lambda: Serviço de computação Serverless (sem servidor) orientado a eventos. Você faz o upload do seu código (Node.js, Python, Java) e o Lambda o executa em resposta a gatilhos (como a chegada de uma foto no S3). Você só paga pelo tempo de computação consumido (milissegundos).
  • VPC (Virtual Private Cloud): O serviço de rede lógica isolada onde seus recursos da AWS residem. Você controla sub-redes (públicas e privadas), tabelas de roteamento e gateways de rede.

2. Microsoft Azure

A nuvem da Microsoft tem forte apelo em organizações que já dependem pesadamente do ecossistema corporativo Microsoft (Windows Server, SQL Server, .NET, Active Directory).

Principais Serviços Correspondentes:

  • Azure Virtual Machines: O equivalente direto do EC2 da AWS. Fornece IaaS para implantação de máquinas virtuais Linux e Windows.
  • Azure Blob Storage: O equivalente do Amazon S3. "Blob" significa Binary Large Object. É a solução de armazenamento de objetos da Microsoft para grandes volumes de dados não estruturados.
  • Azure SQL Database: Banco de dados relacional (PaaS) totalmente gerenciado, construído com base na engine do SQL Server da Microsoft.
  • Azure Functions: O equivalente ao AWS Lambda. É a solução Serverless para rodar pequenos trechos de código orientados a eventos.
  • Azure Active Directory (Azure AD / Microsoft Entra ID): O grande trunfo do Azure. Serviço de gerenciamento de identidades e acessos baseado em nuvem, permitindo que empresas sincronizem facilmente as senhas do escritório físico com as senhas da Nuvem e do Microsoft 365.

3. Google Cloud Platform (GCP)

O GCP é reconhecido mundialmente pela sua força bruta no processamento de dados (Big Data), Machine Learning e por ser o berço de criação da tecnologia de orquestração de contêineres Kubernetes.

Principais Serviços Correspondentes:

  • Compute Engine (GCE): O ambiente IaaS do Google para execução de máquinas virtuais na rede de fibra óptica global do Google.
  • Cloud Storage: O equivalente ao S3 e ao Blob Storage. O serviço de armazenamento unificado de objetos do GCP.
  • Cloud SQL e Cloud Spanner: Cloud SQL atua como o RDS para bancos tradicionais (MySQL, PostgreSQL). Já o Spanner é o diferencial de prova: um banco de dados relacional único que escala horizontalmente com consistência global (o que é raríssimo para bancos SQL).
  • Cloud Functions: A resposta do Google ao Lambda e Azure Functions (FaaS/Serverless).
  • BigQuery: Data warehouse corporativo na nuvem (PaaS/SaaS) absurdamente rápido, Serverless, construído para analisar petabytes de dados utilizando a linguagem SQL padrão.
  • Google Kubernetes Engine (GKE): Um dos serviços de orquestração de contêineres gerenciados mais robustos do mercado, dado que o Kubernetes foi criado internamente pelos engenheiros do próprio Google.

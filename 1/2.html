Endereçamento e Protocolos TCP/IP

🌐 Endereçamento IP (IPv4 vs IPv6)

IPv4

  • 🎯 Tamanho: 32 bits (4 bytes).
  • 📝 Formato: Decimal Pontuada (ex: 192.168.1.1).
  • 🔐 IPSec: Opcional.
  • 🔄 Loopback: 127.0.0.0 a 127.255.255.255.

IPv6

  • 🎯 Tamanho: 128 bits (16 bytes).
  • 📝 Formato: Hexadecimal (ex: 2001:0db8::ff00:42:8329).
  • 🔐 IPSec: Nativo (Obrigatório na especificação original).
  • 🔄 Loopback: ::1

⚖️ Camada de Transporte: TCP vs UDP

Característica TCP (Transmission Control Protocol) UDP (User Datagram Protocol)
Conexão Orientado a Conexão (3-Way Handshake) Não Orientado a Conexão
Confiabilidade Alta. Garante a entrega (retransmite falhas). Nula. Não garante entrega (Best Effort).
Velocidade Mais lento (tem muito overhead/controle). Mais rápido (cabeçalho leve de 8 bytes).
Uso Principal Web (HTTP), E-mail (SMTP), Arquivos (FTP). Streaming, VoIP, DNS, Jogos Online.

🚨 Pegadinhas de Prova

  • DNS vs ARP: A banca diz que o ARP converte nome (www) para IP. Falso. Quem traduz Nome para IP é o DNS. O ARP descobre o Endereço Físico (MAC) a partir de um IP.
  • Modelo OSI vs TCP/IP: Protocolos de Aplicação, Apresentação e Sessão do OSI correspondem a UMA ÚNICA camada no TCP/IP: A camada de Aplicação.

1. A Arquitetura TCP/IP

A arquitetura TCP/IP não é apenas uma dupla de protocolos, mas uma pilha (suíte) completa que rege a Internet mundial. Diferente do modelo OSI (que é puramente teórico e possui 7 camadas), o modelo TCP/IP é prático e implementado.

Em provas, o modelo TCP/IP pode ser cobrado com 4 ou 5 camadas (ambas as abordagens estão corretas, dependendo do autor adotado pela banca, como Kurose ou Tanenbaum):

  • 5. Aplicação: Combina as camadas 5, 6 e 7 do OSI. Responsável pelos serviços de rede para os usuários. Ex: HTTP, SMTP, DNS, FTP.
  • 4. Transporte: Comunicação fim a fim. Responsável pela segmentação e, no caso do TCP, pela confiabilidade. Ex: TCP, UDP.
  • 3. Internet (ou Rede): Responsável pelo roteamento de pacotes através da rede lógica. Ex: IP, ICMP, IPSec.
  • 2. Enlace (Link de Dados): Responsável pela entrega do quadro no mesmo segmento de rede (rede local).
  • 1. Física: Transmissão dos bits brutos pelo meio (cabo, ar, fibra).
    *Nota: Em abordagens de 4 camadas, Enlace e Física são fundidas na camada "Acesso à Rede".

2. O Escovão de Bits: Camada de Transporte

A camada de transporte gera as famosas Portas (sockets), essenciais para que um computador saiba para qual aplicativo entregar o dado (Porta 80 vai pro Navegador, Porta 25 vai pro Email).

O Protocolo TCP e o 3-Way Handshake

O TCP é incrivelmente complexo porque ele precisa garantir que o dado chegue na ordem correta e sem perdas em um ambiente caótico (a internet). Para isso, ele estabelece uma conexão formal antes de enviar qualquer byte de dado. Esse processo cai demais em provas avançadas:

  • Passo 1 (SYN): O Cliente envia um pacote com a flag SYN ativa, pedindo para sincronizar (abrir conexão) com o Servidor.
  • Passo 2 (SYN-ACK): O Servidor recebe, concorda e responde com SYN e ACK (Acknowledge - reconhecimento).
  • Passo 3 (ACK): O Cliente confirma o recebimento do Servidor enviando um ACK final. Conexão estabelecida! Os dados agora podem trafegar.

⚠️ Ataque SYN Flood: Ocorre quando um hacker manda milhares de pacotes do "Passo 1" (SYN), mas nunca responde o "Passo 3" (ACK). O Servidor fica esperando infinitamente com portas abertas até travar (Ataque de Negação de Serviço - DoS).

O Protocolo UDP

O UDP é o exato oposto. Ele não faz handshake, não liga para a ordem dos pacotes e não retransmite se algo der errado. Ele simplesmente "cospe" os dados na rede (Datagramas).

Por que usaríamos algo "inseguro"? Por causa da baixa latência (velocidade). Em uma videochamada ou jogo online, é melhor perder um frame de imagem (um pequeno travamento na tela) do que parar tudo para esperar o pacote perdido ser retransmitido (o que geraria um delay insuportável).

3. Protocolos de Internet (Rede)

Protocolo IP (Internet Protocol)

Sua função é fornecer roteamento. Ele entrega o pacote da origem ao destino com base em endereços lógicos. Características de prova: o IP é um protocolo connectionless (não orientado a conexão) e de Best-Effort (melhor esforço). Ou seja, o protocolo IP, por si só, não garante entrega. Quem garante a entrega é o TCP na camada de cima.

ICMP (Internet Control Message Protocol)

Como o IP não é confiável, o ICMP atua como seu "assistente de diagnóstico e reporte de erros". É o protocolo utilizado por trás de comandos famosos como o PING (que manda mensagens de Echo Request / Echo Reply) e o Tracert/Traceroute.

ARP (Address Resolution Protocol)

Vital em provas! Um pacote viaja pela internet usando o IP, mas quando ele chega na rede local de destino, o switch não entende IP. O switch precisa do Endereço MAC (físico) da placa de rede. O ARP grita na rede (em broadcast): "Quem é o dono do IP 192.168.1.5?". O computador dono do IP responde: "Sou eu, meu MAC Address é XX:XX:XX...".
Resumo: ARP converte IP lógico em MAC Address físico.

4. Endereçamento e Sub-redes (Overview Crítico)

Endereços IPv4 de Teste: A banca vai cobrar o endereço de Loopback (Localhost), que serve para a máquina testar a própria pilha TCP/IP. No IPv4, a faixa inteira 127.0.0.0/8 é reservada para isso (geralmente usamos o 127.0.0.1). No IPv6, é o ::1.

APIPA (Automatic Private IP Addressing): Quando seu computador está configurado para receber IP dinâmico (DHCP), mas o servidor DHCP cai ou o cabo desconecta, o Windows atribui um IP automático de falha que começa com 169.254.x.x. Viu esse IP na prova? Significa falha na rede!

CIDR (Roteamento Interdomínios Sem Classe): O antigo sistema de classes (A, B, C, D e E) foi abolido na prática e substituído pelo CIDR (ex: /24, /26). O valor após a barra indica quantos bits estão ligados (1) na máscara de sub-rede para representar a rede.
Exemplo em prova: /24 significa que 24 bits representam a Rede, sobrando 8 bits para Hosts. Com 8 bits, podemos ter \(2^8 = 256\) endereços, mas tiramos 2 (um para a Rede e um para o Broadcast), sobrando 254 hosts válidos.

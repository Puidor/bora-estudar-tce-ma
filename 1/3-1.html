Gerenciamento de Redes TCP/IP

🧠 A Sopa de Letrinhas (SNMP, MIB, SMI)

📜 SMI

A Gramática. Define as regras gerais e os tipos de dados permitidos (ex: Inteiro, String). É baseada em ASN.1.

🗄️ MIB

O Dicionário. É o banco de dados hierárquico (em árvore) que armazena as variáveis do dispositivo (ex: temperatura, uso de CPU). Identificado pelos OIDs.

🚚 SNMP

O Transportador. É o protocolo de rede em si que leva a mensagem do Gerente para o Agente, solicitando ou alterando dados da MIB.

🛡️ Evolução do SNMP (v1 x v2c x v3)

MUITO COBRADO
Versão Principais Inovações Segurança
SNMPv1 Get, Set, GetNext, Trap (básico). Nenhuma. Autentica por "Community String" em texto claro.
SNMPv2c Introduz GetBulk (puxa vários dados) e Inform (Trap com recibo ACK). Continua usando Community String em texto claro (Inseguro).
SNMPv3 Foco total na reestruturação de arquitetura segura (USM/VACM). Autenticação forte (Hash) + Criptografia (Privacidade).

🚪 Portas UDP do SNMP

  • Porta 161 (Agente): Onde o Roteador/Switch fica escutando as ordens (Get/Set) enviadas pelo Gerente.
  • Porta 162 (Gerente): Onde o Gerente fica escutando. Usada exclusivamente para receber os Traps (Avisos de perigo/erro que o agente manda sozinho).

🚨 Pegadinhas de Prova

  • Trap garante entrega? A banca diz que o Trap é confiável. Falso. Trap usa UDP, se o pacote sumir, o gerente nunca saberá. A solução para isso é o InformRequest (v2c/v3), que exige um ACK do gerente.
  • SNMPv2c Segurança: A letra "c" do v2c significa Community-based. Ele trouxe melhorias de performance, mas falhou em melhorar a segurança em relação ao v1.

1. A Arquitetura de Gerenciamento de Redes

O modelo tradicional de gerenciamento de redes TCP/IP é baseado no paradigma Gerente-Agente. Ele é composto por três peças fundamentais:

  • Gerente (NMS - Network Management System): É o "chefe". Um software rodando em um servidor central (ex: Zabbix, PRTG, Nagios). Ele é responsável por enviar as solicitações de leitura ou alteração de configurações. Ele atua ativamente realizando "Polling" (perguntando aos dispositivos de tempos em tempos) e passivamente aguardando alertas emergenciais.
  • Agente (Agent): É o "funcionário". Um pequeno módulo de software que roda dentro do dispositivo de rede (como um roteador da Cisco, um switch ou uma impressora). Sua função é ficar monitorando o hardware local, respondendo ao Gerente e avisando o Gerente imediatamente caso algo dê errado (via Trap).
  • Objeto Gerenciado: É a informação específica dentro do agente que estamos monitorando (ex: status da porta Gigabit 0/1, temperatura da CPU, tráfego de entrada).

2. Estruturação dos Dados: SMI e MIB

Para que um servidor Dell nos EUA entenda a temperatura enviada por um switch Huawei na China, é necessário haver uma padronização universal dos dados. É aqui que entram a SMI e a MIB.

SMI (Structure of Management Information)

A SMI define a sintaxe universal. Ela dita as regras sobre como os objetos gerenciados devem ser nomeados e quais tipos de dados eles podem assumir. Ela é fortemente baseada em um subconjunto da linguagem ASN.1 (Abstract Syntax Notation One).
Exemplo em prova: A SMI garante que uma variável de "velocidade de rede" seja sempre lida como um número inteiro por qualquer fabricante.

MIB (Management Information Base)

A MIB é o banco de dados virtual que reside no Agente. Ela possui uma estrutura em forma de árvore (hierárquica), semelhante a pastas no Windows.

Cada objeto dentro da MIB possui um endereço único chamado de OID (Object Identifier). O OID é uma sequência de números separados por pontos (ex: 1.3.6.1.2.1.2.2.1).

  • 1 = ISO
  • 3 = Organização identificada
  • 6 = Departamento de Defesa (DoD)
  • 1 = Internet...

3. Operações do Protocolo SNMP

O SNMP atua na Camada de Aplicação e, por questões de performance e simplicidade (para não sobrecarregar processadores antigos de roteadores), ele utiliza o UDP (User Datagram Protocol) na camada de transporte.

Comandos do GERENTE ➔ AGENTE

  • GetRequest: Busca o valor de uma variável específica.
  • GetNextRequest: Busca o valor da próxima variável na tabela MIB.
  • GetBulkRequest (v2c/v3): Pede grandes blocos de dados de uma vez só (alta eficiência).
  • SetRequest: Altera a configuração do agente (ex: desliga uma porta).

Comandos do AGENTE ➔ GERENTE

  • Response: A resposta aos comandos "Get" e "Set".
  • Trap: Um alerta não solicitado. O roteador reiniciou? Ele manda um Trap avisando. Como é UDP e não exige confirmação, pode se perder na rede.
  • InformRequest (v2c/v3): O "Trap Seguro". É um alerta que exige que o Gerente responda com um Response (recibo). Se o recibo não chegar, o agente envia de novo.

4. A Trilha de Segurança do SNMP

SNMPv1 e v2c (Community Strings): Nestas versões, a autenticação baseia-se num conceito de "Community String", que funciona como uma senha compartilhada (geralmente `public` para leitura e `private` para escrita). O problema fatal para a segurança é que essas strings trafegam em texto claro pela rede. Qualquer atacante fazendo sniffing captura a senha instantaneamente.

SNMPv3 (A Revolução de Segurança): Para mitigar esses problemas, o SNMPv3 abandonou as "communities" e implementou dois novos módulos de segurança robustos:

  • USM (User-based Security Model): Responsável pela autenticação e privacidade. Permite autenticar os usuários com HMAC-MD5 ou HMAC-SHA e criptografar o pacote inteiro (Payload) com algoritmos como DES e AES.
  • VACM (View-based Access Control Model): Controle de acesso minucioso. Permite configurar que o "Usuário João" pode ler a MIB de tráfego de rede, mas não tem permissão para ler ou alterar a MIB de roteamento.
Níveis de Segurança do SNMPv3 (Caem muito!):
  1. noAuthNoPriv: Sem autenticação criptográfica e sem criptografia (similar ao v1/v2c).
  2. authNoPriv: Autenticação ativada (via Hash, garantindo quem enviou e integridade), mas pacote trafega sem criptografia (conteúdo legível).
  3. authPriv: O nível máximo. Autenticado (Hash) e Criptografado (AES/DES). Confidencialidade e Integridade totais.

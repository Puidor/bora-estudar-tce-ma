☁️ Modelos de Serviço (A Pirâmide SPI)

IaaS (Infraestrutura)

Máquinas virtuais, redes e armazenamento "crus". Você instala e gere o Sistema Operativo.

Ex: AWS EC2, Azure VMs.

PaaS (Plataforma)

Foco em Programadores. A nuvem gere o SO e os servidores; você gere apenas a sua aplicação e os dados.

Ex: Heroku, Google App Engine.

SaaS (Software)

Foco no Utilizador Final. Software pronto a usar via browser. O fornecedor gere tudo (da rede à aplicação).

Ex: Gmail, Microsoft 365, Dropbox.

💻 Virtualização: O Hypervisor (VMM)

O Hypervisor é a camada de software que permite criar e gerir as Máquinas Virtuais (VMs).

Tipo de Hypervisor Como Funciona? Performance / Uso
Tipo 1 (Bare-Metal) Instalado DIRETAMENTE sobre o hardware físico (não existe um SO convencional por baixo). Alto Desempenho. Usado em Datacenters. (Ex: VMware ESXi, Hyper-V, Xen).
Tipo 2 (Hosted) Instalado como uma aplicação sobre um Sistema Operativo hospedeiro (Windows/Linux/macOS). Menor desempenho. Usado em PCs domésticos/testes. (Ex: VirtualBox, VMware Workstation).

📋 As 5 Características (NIST)

  • 1. Autoatendimento a pedido (On-demand)
  • 2. Amplo acesso à rede
  • 3. Agrupamento de recursos (Resource Pooling)
  • 4. Rápida elasticidade
  • 5. Serviço medido (Pay-as-you-go)

🚨 Pegadinhas de Prova

  • Elasticidade vs Escalabilidade:
    👉 Escalabilidade: Capacidade de suportar o aumento de carga adicionando mais hardware (processo geralmente planeado/estático).
    👉 Elasticidade: É a capacidade de crescer e encolher de forma rápida e automática (dinâmica) conforme a procura.
  • Cloud é Virtualização? Falso! A virtualização é a tecnologia que abstrai o hardware. Cloud Computing é o modelo de negócio e entrega de serviços através da internet (que tipicamente usa a virtualização como base).

1. A Essência da Computação em Nuvem

A Computação em Nuvem (Cloud Computing) é um modelo que permite o acesso ubíquo, conveniente e a pedido, através da rede, a um conjunto partilhado de recursos de computação configuráveis (redes, servidores, armazenamento, aplicações e serviços) que podem ser rapidamente aprovisionados e libertados com um esforço mínimo de gestão.

A Definição Oficial (NIST SP 800-145)

Todas as bancas de concurso baseiam-se no documento do NIST, que define 5 características essenciais da Nuvem:

  • 1. Autoatendimento a Pedido (On-demand self-service): O utilizador pode aprovisionar recursos (como tempo de CPU ou espaço em disco) unilateralmente e de forma automática, sem necessitar de interação humana com o prestador do serviço.
  • 2. Amplo Acesso à Rede (Broad network access): Os recursos estão disponíveis através da rede e podem ser acedidos através de mecanismos padronizados por várias plataformas (telemóveis, tablets, portáteis, workstations).
  • 3. Agrupamento de Recursos (Resource pooling): O modelo multi-tenant (multilocatário). Os recursos do fornecedor são agrupados para servir múltiplos consumidores, com recursos físicos e virtuais atribuídos dinamicamente (o cliente não sabe a localização exata do hardware).
  • 4. Rápida Elasticidade (Rapid elasticity): Capacidade de escalar rapidamente para cima (aumentar) ou para baixo (diminuir/libertar recursos) em sintonia com a procura, dando a ilusão de recursos infinitos.
  • 5. Serviço Medido (Measured service): "Pay as you go". O sistema controla, otimiza e reporta a utilização dos recursos de forma transparente, cobrando apenas pelo que foi efetivamente utilizado (gigabytes armazenados, horas de processamento).

2. Modelos de Implantação (Deployment Models)

Onde reside a infraestrutura e quem tem acesso a ela?

  • Nuvem Pública (Public Cloud): A infraestrutura pertence a uma organização que vende serviços na cloud (AWS, Azure, Google Cloud) e é disponibilizada ao público em geral. Maior risco de segurança aparente (ambiente partilhado), mas custos muito baixos.
  • Nuvem Privada (Private Cloud): A infraestrutura é operada exclusivamente para uma única organização (pode ser gerida pela própria ou por terceiros, on-premise ou off-premise). Garante controlo total e isolamento dos dados.
  • Nuvem Híbrida (Hybrid Cloud): Uma composição de duas ou mais nuvens distintas (privada e pública) ligadas por tecnologia padronizada. Exemplo de prova: Cloud Bursting. A empresa usa a sua Nuvem Privada para a operação normal, mas se houver um pico súbito de tráfego na Black Friday, a infraestrutura "transborda" temporariamente e passa a alugar capacidade da Nuvem Pública.
  • Nuvem Comunitária (Community Cloud): A infraestrutura é partilhada por várias organizações que partilham preocupações comuns (ex: um grupo de hospitais, ou departamentos do mesmo Governo).

3. Modelos de Serviço (IaaS, PaaS, SaaS)

Determina o grau de responsabilidade técnica: o que o fornecedor gere vs o que o cliente tem de gerir.

IaaS
Infraestrutura como Serviço

O fornecedor entrega o hardware cru, armazenamento, redes e máquinas virtuais.

Responsabilidade do Cliente:

Instalar o Sistema Operativo (SO), aplicar patches, instalar as bases de dados e a aplicação.

Ex: AWS EC2, Azure Virtual Machines.

PaaS
Plataforma como Serviço

O fornecedor entrega a máquina já com o SO, linguagens de programação e base de dados instalados. Ideal para Programadores.

Responsabilidade do Cliente:

Apenas focar em escrever e enviar o código da sua Aplicação.

Ex: AWS Elastic Beanstalk, Google App Engine.

SaaS
Software como Serviço

O fornecedor gere absolutamente tudo. O cliente apenas acede ao software pronto a usar através de um navegador web.

Responsabilidade do Cliente:

Configurações de utilizadores e os dados inseridos.

Ex: Gmail, Microsoft 365, Salesforce.

4. Virtualização (A Base da Nuvem)

A Nuvem não existiria sem a virtualização. A peça central desta tecnologia é o Hypervisor (Monitor de Máquina Virtual - VMM). É o software que cria, gere e fatia os recursos físicos (CPU, RAM) para as Máquinas Virtuais (VMs).

Tipos de Hypervisor

  • Tipo 1 (Bare-Metal ou Nativo): O Hypervisor roda diretamente sobre o hardware puro. Não há um Sistema Operacional base. Ele É o SO. Alta performance.
    Exemplos: VMware ESXi, Microsoft Hyper-V, XenServer.
  • Tipo 2 (Hospedado / Hosted): O Hypervisor é um software que se instala por cima de um Sistema Operativo hospedeiro normal (Windows, Linux, macOS). Mais lento, pois passa por mais camadas.
    Exemplos: VMware Workstation, Oracle VirtualBox.

Serviços de Diretório: AD e LDAP

⚡ Direto ao Ponto

  • Serviço de Diretório: É um banco de dados hierárquico, altamente otimizado para leituras e pesquisas (e não para escritas constantes), usado para armazenar informações sobre utilizadores, computadores e recursos da rede.
  • LDAP (Lightweight Directory Access Protocol): É o protocolo de comunicação padrão da indústria (Porta TCP/UDP 389) usado para consultar e modificar os serviços de diretório.
  • Active Directory (AD DS): É a implementação proprietária da Microsoft de um serviço de diretório. Utiliza o LDAP para consultas, o Kerberos para autenticação e depende criticamente do DNS para localização de serviços.

🌳 Estrutura Lógica do Active Directory

Hierarquia clássica cobrada em provas (da menor para a maior):

1. Objeto

A unidade básica. Pode ser um Utilizador (João), um Computador (PC-RH) ou uma Impressora.

2. UO (Unidade Organizacional)

A "pasta". Agrupa objetos para facilitar a administração e aplicar políticas (GPOs). Ex: UO_Financas.

3. Domínio

A fronteira central de segurança e administração. Partilha a mesma base de dados. Ex: empresa.local.

4. Árvore e Floresta

Árvore: Domínios com o mesmo espaço de nomes contíguo. Floresta: O nível máximo, agrupa várias árvores distintas que partilham o mesmo Catálogo Global.

🔤 Sintaxe LDAP (Decoreba)

O LDAP utiliza o Distinguished Name (DN) para identificar o caminho exato de um objeto:

CN=Joao Silva, OU=TI, DC=empresa, DC=com
  • CN (Common Name): Nome comum (o objeto em si, ex: pessoa ou PC).
  • OU (Organizational Unit): A Unidade Organizacional (pasta).
  • DC (Domain Component): Os pedaços do domínio DNS. (empresa.com vira DC=empresa, DC=com).

🚨 Pegadinhas de Prova

  • LDAP Autentica? A banca diz que "O AD utiliza o LDAP como o seu principal protocolo de autenticação segura."
    Falso. O LDAP é para consultas/pesquisas. O protocolo principal de autenticação do AD é o Kerberos (embora o LDAP possa ser usado de forma secundária/simples).
  • Lógico vs Físico: "Os Sites e as UOs formam a estrutura lógica do AD."
    Falso. As UOs, Domínios e Florestas são a estrutura Lógica. Os Sites (localidades geográficas ligadas por sub-redes IP) e os Controladores de Domínio (DC) formam a estrutura Física.

1. O Que é um Serviço de Diretório?

Um serviço de diretório é um repositório centralizado e hierárquico. Ao contrário de uma base de dados relacional tradicional (como SQL Server ou Oracle), que é otimizada para lidar com milhares de transações de escrita por segundo, o serviço de diretório é fortemente otimizado para leituras e pesquisas em alto volume.

Ele gere identidades. Quando você faz login no seu computador corporativo, a máquina consulta o diretório para verificar se o seu utilizador existe, se a palavra-passe está correta e a quais pastas e impressoras você tem acesso.

* O padrão avô de todos os serviços de diretório é o X.500 (criado pela ITU-T). Era muito pesado e operava na camada de aplicação do modelo OSI.

2. O Protocolo LDAP (Lightweight Directory Access Protocol)

Como o padrão X.500 era muito "pesado" para os computadores antigos se ligarem, criou-se o LDAP (a versão "Lightweight" / leve), que funciona diretamente sobre TCP/IP.

Características Técnicas para Concursos

  • Portas Padrão: O tráfego LDAP regular ocorre nas portas TCP 389 e UDP 389. O tráfego seguro e encriptado (LDAPS - LDAP over SSL/TLS) ocorre na porta TCP 636.
  • DIT (Directory Information Tree): É a estrutura em árvore como a informação é organizada. Tudo nasce de uma raiz (Root) e ramifica-se.
  • Esquema (Schema): Define as regras do diretório. Diz, por exemplo, que o objeto do tipo "Utilizador" DEVE ter o atributo nome e PODE ter o atributo telefone. Se quiser adicionar a data de aniversário da pessoa e isso não existir, terá que "estender o esquema".
  • DAP (Directory Access Protocol): O LDAP define um conjunto de operações para interagir com o diretório: Bind (conectar/autenticar), Search (pesquisar), Add, Delete, Modify e Unbind (desconectar).

3. Microsoft Active Directory (AD DS)

O Active Directory Domain Services é a principal implementação de diretório do mercado.

A Trindade do AD

Para o AD funcionar, ele depende vitalmente de três componentes tecnológicos:

  1. LDAP: Para que os clientes e serviços consultem o diretório.
  2. DNS (Domain Name System): Fundamental! O AD utiliza registos DNS especiais (registos SRV) para que um computador saiba qual é o IP do Controlador de Domínio mais próximo. Sem DNS, o AD não funciona.
  3. Kerberos: O protocolo de autenticação principal. Baseia-se em bilhetes (Tickets) e num servidor de confiança (KDC - Key Distribution Center) que roda no Controlador de Domínio. Funciona na porta TCP/UDP 88.

Estrutura Física: Sites e Catálogo Global

  • Controlador de Domínio (DC): O servidor real que aloja a base de dados do AD (o ficheiro NTDS.DIT). Nas versões modernas do Windows, todos os DCs são pares (Multi-master replication), ou seja, se alterar uma senha no DC-1, ele replica para o DC-2.
  • Sites: Representam a topologia física da rede (ex: Escritório de Lisboa, Fábrica no Porto). São mapeados por Sub-redes IP. Servem para otimizar a replicação de dados e garantir que um utilizador em Lisboa faça login no servidor de Lisboa (sem atravessar o link WAN lento até o Porto).
  • Catálogo Global (GC - Global Catalog): Funciona como o "Índice" da Floresta. Ele contém uma réplica completa de todos os objetos do seu próprio domínio, e uma réplica parcial (apenas atributos principais) de todos os objetos dos outros domínios da floresta. Responde na porta TCP 3268. Essencial para buscas rápidas em redes globais gigantes.

4. Interoperabilidade (Windows ⇄ Linux)

A interoperabilidade é a capacidade de sistemas diferentes "conversarem". Num ambiente corporativo real, temos servidores Linux e estações Windows, e queremos que o utilizador use a mesma palavra-passe (Single Sign-On / Integração) em ambos.

Como colocar o Linux no Domínio AD?

Para o Linux entender o AD, ele precisa falar os mesmos protocolos (Kerberos e LDAP). Existem várias ferramentas de mercado que fazem essa ponte. As bancas cobram as seguintes tecnologias:

  • SAMBA (Winbind): O Samba nasceu para partilhar ficheiros (protocolo SMB/CIFS), mas a ferramenta winbind permite que o Linux resolva nomes e grupos perguntando ao Active Directory. O Samba versão 4 e superiores também consegue atuar como se fosse um Controlador de Domínio do próprio AD!
  • SSSD (System Security Services Daemon): A ferramenta moderna padrão em distribuições como Red Hat e CentOS para conectar o Linux ao AD (ou a outros servidores LDAP). Ele faz cache de credenciais localmente, permitindo login mesmo se a rede cair temporariamente.
  • FreeIPA: É uma solução completa de gestão de identidades para o ecossistema Linux (patrocinada pela Red Hat). Em vez de tentar "imitar" o AD, ela pode estabelecer um Trust (Relação de Confiança) com o AD da Microsoft.
  • PAM (Pluggable Authentication Modules): É o coração da autenticação no Linux. É a arquitetura que permite "plugar" módulos de autenticação. É através do PAM que configuramos o sistema operativo Linux para, em vez de olhar para o ficheiro /etc/shadow local, enviar o pedido de autenticação para o SSSD ou Winbind.

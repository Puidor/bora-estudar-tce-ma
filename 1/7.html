🧠 O Núcleo: Monolítico vs Microkernel

Monolítico

  • ✓ Vantagem: Alta Performance. Tudo roda no mesmo espaço de memória (Modo Kernel).
  • ✗ Desvantagem: Se um driver falhar (crash), o sistema inteiro congela (Tela Azul/Kernel Panic).
  • Exemplos: Linux, DOS, Sistemas Unix clássicos.

Microkernel

  • ✓ Vantagem: Alta Estabilidade. Serviços rodam separados no Espaço do Usuário. Se um driver cai, o resto continua.
  • ✗ Desvantagem: Mais Lento devido à troca constante de mensagens (IPC).
  • Exemplos: Minix, QNX. (Windows e macOS são híbridos).

Processo vs Thread

Processos são pesados e NÃO compartilham memória entre si. Threads (fios de execução) são leves e COMPARTILHAM a mesma memória e recursos do processo pai.

Deadlock (Abraço Mortal)

Ocorre quando o Processo A prende o recurso 1 e pede o 2. O Processo B prende o 2 e pede o 1. Ambos ficam travados infinitamente.

🚨 Pegadinhas de Gestão de Memória

  • Fragmentação:
    👉 Paginação gera fragmentação INTERNA (sobra espaço inútil dentro da página fixa).
    👉 Segmentação gera fragmentação EXTERNA (sobra espaço inútil entre os segmentos na memória).
  • Swapping (Páginação em Disco): A banca diz que "O uso intenso de memória virtual/swapping melhora o desempenho geral da máquina".
    Falso! O acesso ao disco rígido é milhares de vezes mais lento que a memória RAM. Se o SO passar a vida toda trocando páginas com o disco (fenômeno chamado de Thrashing), a máquina fica extremamente lenta.

1. Fundamentos e Arquitetura do Kernel

O Sistema Operacional atua como um gerenciador de recursos e uma máquina estendida. Ele abstrai a complexidade do hardware cru (discos, CPU, placa de rede) entregando APIs e interfaces fáceis para os programas e usuários (o conceito de Máquina Virtual/Estendida).

Modo Usuário vs Modo Kernel

Os processadores modernos possuem níveis de privilégio de hardware (Anéis / Rings).

  • Modo Usuário (Ring 3): Onde rodam os programas comuns (Navegador, Word). Não têm acesso direto ao hardware. Se um programa tenta fazer algo proibido, o SO encerra-o.
  • Modo Kernel (Ring 0): Privilégio total e absoluto. Controla a memória, processos e fala com o hardware. Para um programa pedir para ler o disco rígido, ele precisa invocar uma System Call (Chamada de Sistema), mudando temporariamente para o Modo Kernel.

2. Gestão de Processos e Deadlocks

Um Processo é um programa em execução. Ele possui um código, dados, pilhas (stacks) e um bloco de controle (PCB - Process Control Block).

Escalonamento (Scheduling)

Num PC com 1 processador de núcleo único, apenas 1 processo roda de cada vez. O Escalonador (Scheduler) gera a ilusão de paralelismo trocando a CPU de processo rapidamente.

  • Escalonamento Cooperativo: O processo só sai da CPU se ele quiser ou quando terminar. Problema: Um programa travado trava a máquina inteira (comum no Windows 3.1).
  • Escalonamento Preemptivo: Padrão atual. O SO usa uma interrupção de relógio. Se o "Quantum" (tempo na CPU) acabar, o SO arranca o processo à força e coloca o próximo. (Ato chamado de Troca de Contexto / Context Switch).

Deadlock (Atenção às 4 Condições)

O interbloqueio ocorre apenas se QUATRO condições acontecerem simultaneamente (caem com frequência na FCC/FGV):

  1. Exclusão Mútua: O recurso só pode ser usado por 1 processo por vez.
  2. Posse e Espera: O processo retém um recurso enquanto espera por outro.
  3. Não Preempção: O recurso não pode ser tirado à força do processo.
  4. Espera Circular: P1 espera por P2, que espera por P3... que espera por P1.

3. Gestão de Memória

Sistemas modernos usam o conceito de Memória Virtual. Ela separa os endereços que o programa vê (Endereços Lógicos) da RAM física real, além de usar o Disco Rígido como extensão da RAM (Swapping). A conversão lógico/físico é feita na CPU por um chip chamado MMU (Memory Management Unit).

Paginação vs Segmentação

  • Paginação: A memória (lógica e física) é dividida em blocos de tamanho rigorosamente fixo (geralmente 4KB).
    -> Elimina fragmentação externa, mas gera fragmentação interna (se o programa só precisa de 1KB, os outros 3KB da página são desperdiçados).
  • Segmentação: A memória é dividida em blocos de tamanho variável conforme a necessidade da estrutura do programa (ex: um segmento para código, outro para pilhas).
    -> Não gera fragmentação interna, mas gera forte fragmentação externa (buracos entre os segmentos após serem apagados).

4. Entrada e Saída (I/O)

Transferir dados do disco para a memória exige coordenação. Como a CPU é notificada de que o disco terminou de buscar o arquivo?

  • Espera Ocupada (Polling): A CPU fica constantemente a perguntar ao dispositivo: "Já acabou? E agora, já?". Desperdiça tempo total da CPU.
  • Interrupção (Interrupt): O disco avisa: "Cpu, pode fazer as suas coisas. Quando eu terminar, envio-te um sinal (IRQs)". Muito mais eficiente.
  • DMA (Direct Memory Access): Essencial em provas! Um chip especial transfere dados massivos (ex: 1GB) do Disco rígido DIRETAMENTE para a Memória RAM, sem que a CPU precise tratar byte a byte. Poupa a CPU para outras tarefas enquanto os dados viajam.

5. Windows Server e Linux (Diferenciais Administrativos)

Windows Server

A joia da coroa corporativa do Windows Server é o Active Directory (AD DS). Ele funciona como o catálogo global da rede e o centro de identidade. Em conjunto com a GPO (Group Policy Object), permite a um administrador impor de forma centralizada que nenhum computador da rede possa usar pendrives ou trocar de papel de parede. O sistema de ficheiros nativo de alta segurança é o NTFS, com transição atual para o ReFS em grandes volumes de dados.

Linux (Sistemas Unix-like)

O Linux é estruturado de forma completamente diferente. A principal filosofia é: "Tudo é um arquivo" (processos, dispositivos de hardware, conexões de rede... tudo é mapeado no sistema de ficheiros virtual como `/dev` ou `/proc`).

Estrutura de Permissões: Caem muito em provas comandos como chmod, chown. O esquema padrão possui três blocos de leitura/escrita/execução (rwx):
1º bloco: O Utilizador Dono (User).
2º bloco: O Grupo (Group).
3º bloco: Todos os outros (Others).

Exemplo de prova: Um arquivo com permissão 777 dá acesso total a toda a gente, um enorme risco de segurança. O usuário administrador supremo no Linux tem sempre o nome root.
